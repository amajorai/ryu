---
title: "Sandboxing and isolation"
description: "Swappable sandbox backends for tool execution, and what isolation each one actually provides today."
---

Tools that execute code run inside a sandbox backend. Ryu treats the sandbox as swappable - you can pick a default and override it per call - so the isolation strength is a deployment choice rather than a hardcoded one. For the Core-side configuration surface, see [Sandbox](/docs/core/sandbox).

## Backends

| Backend | Selection | Isolation today |
|---|---|---|
| Default in-process / local exec | Default, with per-call override | Process-level; relies on command approval and env scrubbing for safety |
| microsandbox | Detected if its CLI is present | Detection only - wired to recognize availability, not yet the execution path |
| opensandbox | Detected if its CLI is present | Detection only - same as above |
| secure-exec bridge | Opt-in | Linux-only, unverified |

The default backend is the one that actually executes today. The non-default backends are detected so the UI can show what is installed, but selecting them does not yet route execution through them.

<Callout type="warn">
The secure-exec bridge is Linux-only and unverified - do not rely on it for isolation on macOS or Windows, and treat its Linux path as experimental. microsandbox and opensandbox are detection-only at this stage - their presence is recognized but execution still flows through the default backend. Until a stronger backend is the real execution path, your isolation is whatever the default backend plus [command approval](/docs/security/command-approval) and [environment scrubbing](/docs/security/credential-scrubbing) give you. Plan accordingly.
</Callout>

## Why this matters

Because a tool call originates from untrusted model output, the sandbox is the containment layer for code the model decided to run. Where the backend is weak (the default in-process path), the compensating controls are command approval (a human gates dangerous shell commands) and environment scrubbing (the child never inherits your secrets). Treat those two as load-bearing, not optional.
