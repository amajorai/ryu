A Ryu node is one running Core instance ([`apps/core`](/docs/start-here/architecture/core-vs-gateway), `:7980`).
This page is the reference for the three node-level surfaces that answer "who and what is on this node":
connected-client **presence**, the optional **mesh** (Tailscale/Headscale) that links nodes privately, and
the public **webhook ingress** seam. The desktop renders all three in the node dropdown (`NodeSelector`).

<Callout type="warn">
Presence is **attribution, not verified identity**. Client identities are self-declared behind the shared
`RYU_TOKEN` and carry no privilege. The data model is single-tenant (conversations and memory have no
`user_id`), so presence answers "who is here", never "who may see what". See the caveats below.
</Callout>

## Connected-client presence

The presence registry (`apps/core/src/connections/mod.rs`, `ConnectionRegistry`) is an in-memory, TTL-based
view of the clients currently talking to a node. A `track_connection` middleware (`apps/core/src/server/mod.rs`)
runs **inside** `require_auth`, so only authenticated protected requests are counted (public routes like
`/api/health` never pollute it). Each request parses a self-declared identity from headers and `touch`es the
registry with the current time; entries age out after the TTL with no persistence.

### Identity headers

Clients declare who they are with these headers. The desktop sends them on every Core request from one place
(`apps/desktop/src/lib/api/client.ts`, `identityHeaders()`).

| Header | Field | Role |
|---|---|---|
| `x-ryu-client-id` | `client_id` | Stable per-install id (random, persisted client-side). The **dedup key**, required to be tracked. |
| `x-ryu-client-label` | `client_label` | Short device label, for example `Desktop`, `CLI`, `Phone`. |
| `x-ryu-surface` | `surface` | Surface kind, for example `desktop`, `cli`, `mobile`, `extension`. |
| `x-ryu-user-id` | `user_id` | Stable per-user id (URL-encoded; the desktop sends the control-plane email). |
| `x-ryu-user-name` | `user_name` | Human display name (URL-encoded so non-ASCII names survive). |

A client with no `x-ryu-client-id` is not trackable and stays invisible. Older clients that send no identity
headers still work; they just do not appear.

### Reading presence

```
GET /api/connections
```

Returns the active set (newest activity first) with rollup counts:

| Field | Meaning |
|---|---|
| `data` | Array of connected clients, each with `client_id`, `user_id`, `user_name`, `client_label`, `surface`, `first_seen`, `last_seen`. |
| `client_count` | Number of distinct clients within the TTL window. |
| `user_count` | Distinct declared users (an anonymous client counts once by `client_id`). |
| `ttl_secs` | The TTL window in seconds (`DEFAULT_TTL_SECS`, currently `90`). |

Dedup is by `(user_id, client_id)`, so one user with a desktop and a phone shows as two clients. A re-touch
updates `last_seen` but never `first_seen`; entries last seen more than `ttl_secs` ago are pruned on the next
read. The desktop surfaces this as a "Connected, N" section in `NodeSelector` and marks the current install
"(you)".

<Callout type="warn">
This closes the analog of MCP "enterprise-managed auth" only at the **presence** layer. Identities are
self-asserted behind the shared token, grant no isolation, and real people sharing a node still see each
other's chats. A truly verified tier is deliberately out of scope: verifying a token against the control
plane on every request would break Core's local-first and headless-first rule. A future verified tier would
key off the tailnet peer identity, not the control plane.
</Callout>

## Mesh (Tailscale / Headscale)

The mesh links nodes over a private tailnet so a client can reach a remote node by its MagicDNS name instead of
a LAN IP. Core owns **what runs** here: the daemon is an opt-in Sidecar (`apps/core/src/sidecar/tailscale.rs`,
`TailscaleManager`), never in `startup_order`, using userspace networking plus a SOCKS5 proxy. The read side
(`apps/core/src/mesh/mod.rs`) shells out to `tailscale status --json` and shapes it into one contract.

```
GET /api/mesh/status
```

| Field | Meaning |
|---|---|
| `enabled` | Whether the mesh is enabled on this node (reflects `RYU_MESH_ENABLED`, not the daemon). |
| `reachable` / `up` | Both present and equal; `true` only when `backend_state == "Running"`. |
| `backend` | `tailscale`, `headscale`, or `null` (classified from the control server URL). |
| `backend_state` | Raw `BackendState` string, for example `Running`, `NeedsLogin`, `Stopped`. |
| `control_server` | Coordination/login server URL, or `null` when unknown. |
| `magic_dns_name` | This node's MagicDNS name (trailing dot stripped). |
| `tailscale_ips` | This node's tailnet IPs. |
| `peers` | Peer nodes, each with `name`, `host_or_dns`, `magic_dns_name`, `tailscale_ips`, `online`, `os`. |

When the mesh is disabled, `query_status` returns the all-default object with HTTP 200 (never 500), so a
vanilla install reports `enabled:false` cleanly. When enabled but the daemon is absent or erroring, it returns
an enabled-but-unreachable object so the desktop can render an amber "configured but down" state.

<Callout type="warn">
The mesh is **opt-in and off by default**. Enable it with `RYU_MESH_ENABLED` (truthy is anything but empty,
`0`, `false`, or `no`). It is **fail-closed**: Core refuses to start tokenless under mesh
(`enforce_remote_auth`), the gateway admin/exec gates apply `&& !mesh_enabled()`, and the authkey is removed
from the env after the `0600` keyfile write.
</Callout>

## Webhook ingress

The webhook ingress seam (`apps/core/src/webhook_ingress/`) gives Composio triggers a public URL to call. It is
a swappable `WebhookIngress` source so a node can expose Core via its own relay, the mesh Funnel, a tunnel, or
a Ryu-hosted relay, without changing the rest of the stack.

```
GET /api/webhook-ingress/status
```

Returns the active backend `kind`, its resolved `public_url` (or `null`), and `up` (true once a public URL has
been resolved). The backend selector is read and set at `GET`/`POST /api/webhook-ingress/backend`, resolving
from the env override, then the `webhook.ingress.backend` pref, then the default.

| Backend (`IngressKind`) | What it is |
|---|---|
| `OwnRelay` | BYO: the user already exposes Core at a public URL, configured via `RYU_WEBHOOK_INGRESS_URL`. |
| `TailscaleFunnel` | Exposes Core's bind port over the mesh's Tailscale Funnel (needs the mesh enabled). |
| `Cloudflared` | A Cloudflare tunnel in front of Core. |
| `RyuRelay` | Ryu-hosted managed push (outbound SSE client into `composio_triggers`). |

<Callout type="warn">
The public ingress is **HMAC fail-closed**: it rejects inbound webhooks until the signing scheme is verified.
The `RyuRelay` backend is managed push, not a public listener; see the marketplace and Composio trigger paths
for how events fan in.
</Callout>
