---
title: "Core Internals"
description: "The local Rust backend (port 7980) that orchestrates agents, conversations, memory, tools, workflows, and all other subsystems."
---

**Ryu Core** is the local Rust backend running on `127.0.0.1:7980` by default. It owns the **what runs** half of the system: engine processes, conversation storage, long-term memory, tool routing, workflow execution, scheduling, and MCP registry. All clients (desktop, CLI, Island, Extension, mobile) are thin surfaces over the same HTTP API. Every model call passes through the Gateway; see [Core vs Gateway](/docs/start-here/architecture/core-vs-gateway) for where that split lives.

<Callout type="info">
State is stored under `~/.ryu/` - SQLite for conversations, memory, agents, and traces; JSON for workflow runs and scheduled jobs. The data folder location is configurable in Settings under System → Storage.
</Callout>
