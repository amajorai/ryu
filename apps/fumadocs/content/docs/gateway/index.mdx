---
title: "Gateway"
description: "The LLM control plane: request routing, provider management, firewall rules, spend budgets, evals, audit logs, tool governance, and reliability settings."
---

The **Gateway** (`apps/gateway`) is a Rust process that sits between every client and every model
provider. It runs on `127.0.0.1:7981` and exposes an OpenAI-compatible API, so any agent or tool
that speaks that protocol can be pointed at it. Core manages it as a sidecar in the default desktop
setup; you can also run it standalone in front of your own agents.

<Callout type="info">
  Core decides what runs. The Gateway decides what is allowed. That single rule explains where every
  policy belongs - see [Core vs Gateway](/docs/start-here/architecture/core-vs-gateway).
</Callout>
