The [interactive Gateway API reference](/docs/develop/api-reference/gateway) documents every route a normal client should call. A handful of additional routes exist for Core integration, subscription-preserving agent proxies, or convenience aliases. This page explains what they are, who may call them, and why they are not interchangeable with the public `/v1/*` surface.

## Route coverage summary

| Route | Caller | Auth | Documented in |
|---|---|---|---|
| `POST /sandbox/tick` | Core sandbox heartbeat | Trusted-forwarder or master key | [OpenAPI](/docs/develop/api-reference/gateway/audit/sandboxTick) |
| `GET /v1/savings` | Core community-stats beacon (opt-in) | None (public aggregate) | [OpenAPI](/docs/develop/api-reference/gateway/health/getCommunitySavings) |
| `GET /v1/cursor/chat/completions` | Cursor (OpenAI-compat alias) | Bearer API key (when auth enabled) | [OpenAPI](/docs/develop/api-reference/gateway/inference/createCursorChatCompletion) |
| `ANY /passthrough/anthropic/*` | Claude Code (loopback subprocess) | Caller's subscription OAuth bearer | This page (not OpenAPI) |
| `ANY /passthrough/openai-responses/*` | Codex ChatGPT-login (loopback subprocess) | Caller's OAuth bearer + `ChatGPT-Account-ID` | This page (not OpenAPI) |

## Passthrough proxies (loopback-only)

Claude Code and Codex (ChatGPT-login mode) speak native wire formats and authenticate with the user's own subscription credentials. The regular OpenAI-compat chat path cannot govern them without substituting the gateway's provider key.

Instead, the gateway runs a **transparent reverse proxy** (`apps/gateway/src/passthrough/mod.rs`):

| Route | Upstream default | Wire format |
|---|---|---|
| `/passthrough/anthropic/*` | `api.anthropic.com` | Anthropic Messages |
| `/passthrough/openai-responses/*` | `chatgpt.com/backend-api/codex` | OpenAI Responses |

Core points the agent subprocess at these routes via `ANTHROPIC_BASE_URL` or an isolated `CODEX_HOME` provider `base_url`. The proxy applies request-side DLP, forwards the caller's own auth headers unchanged, streams the response through response-side DLP, and emits an audit row.

<Callout type="warn">
  **Loopback-only.** Both passthrough routes return `403 Forbidden` for non-loopback peers. A remote gateway never carries a user's subscription bearer. They are opt-in (off by default) and are **not** part of the standard client API — do not point external integrations at them.
</Callout>

Upstream hosts are overridable via `RYU_PASSTHROUGH_ANTHROPIC_UPSTREAM` and `RYU_PASSTHROUGH_CODEX_UPSTREAM`. For enablement, redaction behavior, and limitations, see [Gateway for any agent — subscription-preserving passthrough](/docs/gateway/gateway-for-any-agent#subscription-preserving-passthrough-claude-code-codex).

## Sandbox metering (`POST /sandbox/tick`)

When Core runs a billed sandbox workspace, its heartbeat loop posts one tick per interval to `{gateway_url}/sandbox/tick` with the elapsed second-delta, provisioned `spec`, and monotonic `tick_index`. The gateway:

1. Accrues the marked-up micro-USD cost for the delta.
2. Debits the org wallet (when `org_id` is present and credits are active).
3. Returns `{ accrued_micro_usd, balance_micro_usd, verdict }` where `verdict` is `continue`, `warn`, `kill_budget`, or `kill_balance`.

Authentication matches other Core→Gateway control calls (`POST /v1/exec/audit`): trusted-forwarder or master key only. Unlike `POST /v1/exec/tool`, sandbox ticks are **not** refused under mesh mode.

Implementation: `apps/gateway/src/api/sandbox.rs`, caller: `apps/core/src/sidecar/sandbox/heartbeat.rs`.

## Community savings (`GET /v1/savings`)

A public, ungated aggregate of compression and cache savings:

```json
{
  "requests": 0,
  "input_tokens": 0,
  "output_tokens": 0,
  "tokens_saved": 0,
  "cache_hit_rate": 0.0
}
```

Core's opt-in community-stats beacon (`apps/core/src/stats_beacon.rs`) reads this endpoint and forwards anonymous totals to the control plane. The gateway exposes the same handler at `GET /savings` (no `/v1` prefix). Unlike `GET /v1/metrics`, this response intentionally omits per-provider maps, quota state, and any secret material.

## Cursor chat alias (`POST /v1/cursor/chat/completions`)

Cursor's "Override OpenAI Base URL" setting already emits the standard OpenAI `/v1/chat/completions` JSON shape. The gateway registers a labelled alias at `/v1/cursor/chat/completions` that dispatches to the same governed chat pipeline — point Cursor at `<gateway>/v1/cursor` instead of `/v1`. A future Cursor-specific protocol translation could diverge here without breaking the primary path.

## Convenience aliases (same handler, alternate path)

Several observability routes register twice for ergonomics:

| Alias | Canonical route |
|---|---|
| `GET /metrics` | `GET /v1/metrics` |
| `GET /savings` | `GET /v1/savings` |
| `GET /health` | `GET /v1/health` |

Only the `/v1/*` forms appear in the interactive OpenAPI reference; the aliases behave identically.
