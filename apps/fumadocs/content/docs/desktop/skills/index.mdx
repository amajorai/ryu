The skills catalog browses the public [skills.sh](https://skills.sh) directory of Agent Skills and
installs them locally. Like the model catalog, all the logic lives in Core
(`apps/core/src/skills_catalog/`), so clients are pure UI over the same endpoints.

## What an Agent Skill is

A skill is a reusable capability described by a `SKILL.md` file (a YAML front-matter block with a
name and description, plus a markdown body). Installed skills load into Core's skill registry from
`~/.ryu/skills/`, where agents can use them.

## Browse

```
GET /api/skills/catalog
```

Search the directory with a `query` and a `limit`, or omit the query to get a featured default. The
featured view runs a handful of seed searches (agent, react, next, typescript, python, security,
database, testing), merges and de-duplicates the results, and sorts by install count. Pass
`installed_only=true` to list what is on disk.

Core uses the same anonymous endpoints the official skills.sh CLI uses. The token-gated `/api/v1`
endpoints are deliberately not used.

## Detail

```
GET /api/skills/catalog/detail?id=owner%2Frepo%2Fslug
```

The detail view downloads the skill package, finds `SKILL.md`, splits its YAML front-matter from the
body, and returns the description, the README body, and the full file list.

## Install

```
POST /api/skills/catalog/install
```

Installing downloads the package, writes `SKILL.md` to `~/.ryu/skills/<slug>.md`, hot-reloads the
skill registry so the skill is usable immediately with no restart, and records provenance in
`~/.ryu/skills-catalog-installed.json`. Any sibling docs are written alongside so references are not
lost.

To list every installed skill and its enabled state, use `GET /api/skills`.

## Author your own skill in-app

You do not have to install from the catalog - you can write a `SKILL.md` from scratch in the desktop
and edit it later, with the same PlateJS editor Spaces pages use. Open the Skills page and choose
**New skill** (from the filter panel or the empty state), or **Edit** an installed skill from its
detail header (`apps/desktop/src/pages/SkillEditorPage.tsx`).

The editor gives you the front-matter as form fields - name, description, allowed tools, and an
**Always on** switch - plus the Plate body for the instructions. Core owns the write and stores each
file at `~/.claude/skills/<id>/SKILL.md` through these routes.

```
POST /api/skills          # create a new skill
PUT  /api/skills/:id      # update (autosaved)
GET  /api/skills/:id/source
```

Editing an existing skill autosaves on a short debounce. Creating a new one is explicit: name it,
press **Create skill**, and the tab moves to that skill's edit route. Saving patches only the
editor-managed front-matter keys and preserves any others, so the catalog's own enabled state (which
the activate toggle owns, not this editor) is never clobbered.

## Version history

Every skill you edit in-app carries server-backed version history - the same **snapshot / diff /
restore** control that pages and workflows use. Versions store the raw `SKILL.md` string, so a
restore is lossless and the diff surfaces front-matter changes, not just body edits.

```
GET  /api/skills/:id/versions
POST /api/skills/:id/versions            # snapshot
GET  /api/skills/:id/versions/:version_id
POST /api/skills/:id/versions/:version_id/restore
```

Open **Version history** in the editor header to name a snapshot, compare any version against the
current source, or restore one. A restore auto-snapshots the current state first, so you can always
step back.

<Callout type="info">
  Version history is stored under `~/.ryu/skill-versions/`, separate from the shared skills
  directory, and capped (older versions are pruned). It applies to skills you author or edit in-app,
  not to the catalog packages themselves.
</Callout>

<Callout type="info">
  Skills here are the catalog-installed `SKILL.md` capabilities loaded by Core. They are distinct from
  the Gateway's skill-injection feature, which prepends prompt snippets to system messages at the
  Gateway. See [The Gateway](/docs/gateway).
</Callout>
