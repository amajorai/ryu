title: "Monitors"
description: "Watch a URL on a schedule and get alerted on price, stock, content, keyword, or uptime changes."
---

A **monitor** watches a URL on a schedule and alerts you when something changes. Open **Monitors**
from the command palette to create and manage them.

## Check types

Each monitor has a check type:

- **Uptime** - is the site up, and how fast does it respond.
- **Keyword** - a substring or regex is present, or absent.
- **Content diff** - the page's normalized text changed (optionally only within a region you
  specify).
- **Price** - capture a number from the page and compare it: changed, above, below, drops by a
  percentage, or rises by a percentage.
- **Stock** - an in-stock phrase appeared.

## How alerts work

Every check compares against the previous **snapshot** and alerts only on a **transition** into the
alert state, so you are notified once when something changes - not on every interval. Each monitor
rides Ryu's scheduler, so checks run on a regular tick without you doing anything.

## Notifications

When a monitor fires, the alert fans out across every channel you have wired up:

- An in-app toast and a native desktop notification.
- A push notification to registered mobile devices.
- A per-monitor webhook (for example Slack or Discord).
- A Telegram message.

<Callout type="info">
  Monitors fetch pages with a swappable backend - a plain HTTP fetch by default, or the Spider
  crawler for pages that need rendering. Alerts are stored and streamed live, so the Monitors page
  shows new alerts as they happen.
</Callout>
