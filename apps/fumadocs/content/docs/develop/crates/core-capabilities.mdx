Every capability in Ryu is owned by a dedicated crate with **zero dependency on `apps/core`**.
Core implements each crate's `*Host` trait and injects the implementation at boot.

## Data & Storage

### ryu-rag

**Path:** `crates/core/rag`

RAG primitive: embedding, reranking, and retrieval.

| Export | Type | Purpose |
|---|---|---|
| `RagProvider` | trait | Abstract RAG backend (embed + rerank + retrieve) |
| `Embedder` | struct | In-process embedding (GGUF via llama.cpp sidecar) |
| `Reranker` | struct | In-process reranking (BAAI bge-reranker) |
| `RetrievalStore` | struct | SQLite-backed cosine merge + top-K rerank |

**Build on it:** Implement `RagProvider` for a new backend (Qdrant, Pinecone, Weaviate). The
crate provides default in-process implementations backed by GGUF models served by a dedicated
llama.cpp `--embeddings` instance.

### ryu-memory

**Path:** `crates/core/memory`

Long-term memory: encrypted, multi-level, scoped recall.

| Export | Type | Purpose |
|---|---|---|
| `MemoryStore` | struct | SQLite-backed encrypted memory with CRUD + recall |
| Scope levels | enum | `user`, `node`, `project` — scoped recall |
| Category / importance / tags | fields | Rich metadata for memory items |

**Build on it:** Extend the scope model or add new recall strategies. Memory is encrypted at
rest via `ryu-crypto`'s `FieldCipher`.

### ryu-spaces

**Path:** `crates/core/spaces`

Named document collections with vector store and GraphRAG.

| Export | Type | Purpose |
|---|---|---|
| `SpaceStore` | struct | Documents, chunks, wiki-link graph |
| Vector store | sqlite-vec | Cosine similarity search |
| Blob store | content-addressed | File attachments per space |
| GraphRAG | wikilink graph | Relationship-aware retrieval |

**Build on it:** Add new retrieval strategies or space types. Spaces are the knowledge bases
that agents ground their responses in.

## Execution & Sandboxing

### ryu-sandbox

**Path:** `crates/core/sandbox`

Sandboxed code execution with swappable backends.

| Export | Type | Purpose |
|---|---|---|
| `run_sandboxed()` | fn | Execute code in a sandbox |
| `SandboxHost` | trait | Abstract sandbox backend |
| `select_backend()` | fn | Pick backend: wasmtime, Docker, microsandbox, Daytona |

**Build on it:** Add new sandbox backends via `select_backend`. The wasmtime backend is the
default (Windows-native, no Docker required).

### ryu-durable

**Path:** `crates/core/durable`

| Export | Type | Purpose |
|---|---|---|

**Build on it:** Implement `DurableEngine` for external backends (Temporal, Restate). The default
is in-process via `FallbackEngine` on the workflow engine.
