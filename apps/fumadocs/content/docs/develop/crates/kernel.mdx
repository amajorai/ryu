These crates form the foundation that all other crates and apps build on.

## ryu-kernel-contracts

**Path:** `crates/core/kernel-contracts`

The single, pure-data definition of the `manifest.json` manifest model.

| Export | Type | Purpose |
|---|---|---|
| `PluginManifest` | struct | Full manifest type |
| `Runnable` | struct | Per-runnable metadata |
| Schema types | structs | All manifest field types |
| Validation rules | fns | Semver, id format, kind validation |

**This is the leaf crate** — zero dependencies on other ryu crates. Core, the SDK, and every
capability crate depend on it for manifest types.

**Build on it:** Add new manifest contract types here. Any change ripples to all consumers.

## Apps-Store Backend Crates

These live under `apps-store/*/backend/` and are consumed by Core as path dependencies.
All have **zero dependency on `apps/core`**.

| Crate | Path | Purpose |
|---|---|---|
| `ryu-recipes` | `apps-store/recipes/backend` | Ghost-OS parity: parameterized, replayable desktop automations |
| `ryu-teams` | `apps-store/teams/backend` | Agent teams: named, ordered agent collections + coordination |
| `ryu-research` | `apps-store/research/backend` | Autoresearch experiment runner |
| `ryu-finetune` | `apps-store/finetune/backend` | Fine-tuning durable state: SQLite job store + adapter catalog |
| `ryu-monitors` | `apps-store/monitors/backend` | Website monitors: price/stock/keyword/content-diff/uptime |
| `ryu-meetings` | `apps-store/meetings/backend` | Meeting notes: record, transcribe, AI notes, mic-in-use detect |
| `ryu-quests` | `apps-store/quests/backend` | Quests: auto-detecting todo list |
| `ryu-clips` | `apps-store/clips/backend` | Clips: agent-native screen capture (Loom/Jam) |
| `ryu-mail` | `apps-store/mail/backend` | Agent Inboxes: email-as-a-service over AWS SES/S3 |

**Build on them:** Each crate defines a `*Host` trait. Core implements it and injects the
implementation. To add a new apps-store capability:

1. Create `apps-store/<name>/backend/` with a `Cargo.toml`
2. Define a `<Name>Host` trait
3. Add the crate as a path dependency of `apps/core`
4. Implement the `*Host` trait in Core's boot sequence
