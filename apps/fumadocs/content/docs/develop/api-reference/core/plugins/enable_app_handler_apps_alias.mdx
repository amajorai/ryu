          returned.


          After a successful enable the manifest's Runnables are activated via

          [`crate::runnable::RunnableRegistry`]. Per-Runnable results are
          included in

          the response so the caller can see which Runnables were registered and
          which

          (if any) encountered a partial failure. A partial failure does NOT
          roll back

          the enable — Core-owned kinds (Agent, Workflow, Tool) are activated;
          kinds

          without a built-in handler (Policy, Engine — Gateway's domain) produce
          an

          observable "no handler" error.
