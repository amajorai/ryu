      - content: >-
          Core today has **no** command-invocation endpoint of its own — plugin
          slash

          commands are surfaced read-only via `GET /api/plugins/contributions`
          and are

          dispatched from the desktop palette, a separate process that cannot
          call the

          in-process `fire_activation_event` fn across the boundary. This
          endpoint is

          that seam: the palette POSTs `{ "event": "onCommand:<id>" }` when it
          runs a

          command, and the gated plugins activate.


          Scoped deliberately to the `onCommand:` prefix so it stays the
          onCommand

          seam and cannot be used to spoof `onStartup`/`onChat`/arbitrary
          events. The

          firing is awaited here (this is not the hot chat path) and is
          idempotent.
