title: "Icon"

---

`Icon` is the single primitive every Ryu surface uses to render a glyph from an **id** rather than a
hardcoded name-to-icon map. Apps register an icon id in their manifest and the shell renders it
through this one component ‚Äî companions, system spaces, and (soon) agent icons and marketplace app
cards all resolve the same way.

```tsx
import { Icon, iconToUrl } from "@ryu/ui/components/icon";
```

## Icon id formats

A single Iconify resolver covers three shapes of id, so you can mix any of them:

| Id shape | Example | Resolves to |
|---|---|---|
| **Iconify / icons0.dev** `prefix:name` | `"lucide:heart"`, `"mdi:home"`, `"tabler:mic"` | That collection's glyph. 150+ sets, 200k+ icons. icons0.dev is Iconify-powered, so its catalog resolves through the same host. |
| **Bare Hugeicons name** | `"ai-image"`, `"mic-01"`, `"shapes"` | Mapped to the `hugeicons:` set (`hugeicons:<name>`). This is how the built-in app manifests author their icons. |
| **Ready image URL or `data:` URI** | `"https://‚Ä¶/logo.svg"`, `"data:image/svg+xml,‚Ä¶"` | Returned unchanged ‚Äî no lookup. |

An id with a colon is treated as `prefix:name`; a bare name (no colon) is assumed to be a Hugeicons
name.

## Usage

```tsx
import { Icon } from "@ryu/ui/components/icon";

// Iconify / icons0.dev prefix:name
<Icon icon="lucide:heart" />

// Bare Hugeicons name ‚Üí hugeicons:mic-01
<Icon icon="mic-01" />

// A ready URL or data: URI is used as-is
<Icon icon="https://example.com/logo.svg" />

// Size and an accessible label
<Icon icon="tabler:mic" size={24} label="Start recording" />
```

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `icon` | `string \| null \| undefined` | ‚Äî | The icon id (any of the three shapes above). A null/empty id renders nothing so callers can fall back to a default glyph. |
| `size` | `number` | `16` | Rendered edge length in px. Drives both the box and the fetched SVG size. |
| `className` | `string` | ‚Äî | Merged onto the rendered `<span>`. |
| `label` | `string` | ‚Äî | Accessible label. Present ‚Üí `role="img"` + `aria-label`. Omitted or `""` ‚Üí the icon is `aria-hidden` (decorative). |

## Theme-aware by default

The glyph is painted with a CSS mask whose `background-color` is `currentColor`, so it inherits the
surrounding text color and flips with light/dark themes automatically ‚Äî the same reason the app's
static Hugeicons glyphs inherit text color. You tint an `Icon` by setting text color on it or an
ancestor, not by passing a color prop:

```tsx
// Follows the theme's foreground color
<span className="text-foreground">
  <Icon icon="lucide:heart" />
</span>

// Tinted red ‚Äî the mask follows currentColor
<Icon icon="lucide:heart" className="text-red-500" />
```

<Callout type="info">
  Because it is a mask, a multi-color icon set flattens to a single-color silhouette. Monochrome and
  line icons (Lucide, Tabler, Hugeicons, MDI, ‚Ä¶) render ideally; reach for those.
</Callout>

## `iconToUrl(icon, opts)`

`Icon` resolves the id through `iconToUrl`, which you can also call directly when you need the URL
itself ‚Äî for example as an `<img src>` or a CSS `background-image`:

```tsx
import { iconToUrl } from "@ryu/ui/components/icon";

iconToUrl("lucide:heart");
// ‚Üí "https://api.iconify.design/lucide/heart.svg"

iconToUrl("mic-01", { size: 24 });
// ‚Üí "https://api.iconify.design/hugeicons/mic-01.svg?width=24&height=24"

iconToUrl("lucide:heart", { color: "#ef4444" });
// ‚Üí "‚Ä¶/lucide/heart.svg?color=%23ef4444"  (a fixed color baked into the SVG)

iconToUrl("");        // ‚Üí null
iconToUrl(undefined); // ‚Üí null
```

The `opts.color` param bakes a fixed color into the server-rendered SVG. It only applies when you
use the returned URL directly ‚Äî the `Icon` component never passes it, because its mask already
inherits `currentColor`. To tint an `Icon`, set text color (see above) rather than reaching for
`color`.

An empty or nullish id returns `null` so callers can branch to a default glyph.

## How apps register an icon

An app's manifest declares its companion surface with an `icon` field. The value is any of the id
shapes above; the desktop shell resolves it through this primitive ‚Äî there is no per-app glyph map to
maintain:

```json title="manifest.json"
{
  "id": "com.example.recorder",
  "name": "Recorder",
  "version": "1.0.0",
  "companion": {
    "label": "Recorder",
    "icon": "mic-01",
    "shortcut": "ctrl+shift+r"
  }
}
```

The built-in app manifests author bare Hugeicons names (`"mic-01"`, `"ai-image"`, `"shapes"`); a
`prefix:name` id or a hosted URL works identically. See
[Build a Desktop Companion](/docs/develop/extensions/desktop-companion) for the full companion
surface.

<Callout type="warn">
  Resolution is **online**. `iconToUrl` points at `https://api.iconify.design`, so rendering a glyph
  by id fetches the SVG from that host. A surface that embeds `Icon` must allow the host in its CSP
  `img-src` (the desktop shell and the whiteboard/canvas apps already do). URL and `data:` ids need
  no network round-trip.
</Callout>
