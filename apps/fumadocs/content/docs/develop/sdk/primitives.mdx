The SDK provides composable primitive clients that give your Runnables access to Ryu's full
capability stack. Every primitive routes through the Gateway — there are no direct provider calls.

## The RunnableContext

Every `run(input, ctx)` receives a `RunnableContext` that carries the primitives:

```typescript
interface RunnableContext {
  gateway: GatewayClient;   // model calls (chat, stream)
  rag?: RagClient;          // retrieval-augmented generation
  memory?: MemoryClient;    // long-term conversation memory
  tts?: TtsClient;          // text-to-speech
  stt?: SttClient;          // speech-to-text
  image?: ImageClient;      // image generation
  durable?: DurableClient;  // durable execution checkpoints
  realtime?: RealtimeClient; // WebSocket/RTC streaming
  engines?: EnginesClient;  // direct engine access
  sessionId?: string;       // Core session ID
  signal?: AbortSignal;     // cancellation
}
```

Primitives are optional — they are only available when the Runnable is running inside Core where
the capability broker has resolved them. When running standalone (`ryu dev`), only `gateway` is
guaranteed.

## GatewayClient

The only path to a model. Every Runnable must use `ctx.gateway` for model calls.

```typescript
// Non-streaming
const result = await ctx.gateway.chat([
  { role: "system", content: "You are a helpful assistant." },
  { role: "user", content: "Hello!" },
]);
console.log(result.content);

// Streaming
for await (const delta of ctx.gateway.stream(messages)) {
  if (delta.content) process.stdout.write(delta.content);
}
```

## RagClient

Embed, search, and rerank documents. The RAG client routes through the Gateway's embedding
endpoint.

```typescript
// Embed text
const embeddings = await ctx.rag.embed(["text1", "text2"]);

// Search a space
const results = await ctx.rag.retrieve({
  spaceId: "my-space",
  query: "deployment steps",
  topK: 5,
});

// Rerank results
const reranked = await ctx.rag.rerank(query, results);
```

## MemoryClient

Long-term conversation memory — recall past context and store new memories.

```typescript
// Recall memories related to a query
const memories = await ctx.memory.recall("what did we discuss about architecture?");

// Store a new memory
await ctx.memory.store({
  content: "The team decided to use WASM for sandboxing",
  metadata: { topic: "architecture", decision: true },
});
```

## TtsClient and SttClient

Text-to-speech and speech-to-text, routed through the Gateway.

```typescript
// Generate speech from text
const audio = await ctx.tts.speak("Hello, world!", { voice: "default" });

// Transcribe audio to text
const transcript = await ctx.stt.transcribe(audioBuffer, { language: "en" });
```

## ImageClient

Image generation, routed through the Gateway.

```typescript
const image = await ctx.image.generate("A sunset over a cyberpunk city", {
  size: "1024x1024",
  model: "dall-e-3",
});
```

## DurableClient

Durable execution checkpoints — save and resume long-running workflows.

```typescript
// Create a checkpoint
const checkpoint = await ctx.durable.checkpoint({
  step: "awaiting_approval",
  data: { proposalId: "123" },
});

// Resume from checkpoint (on next invocation)
if (checkpoint) {
  // Continuing from step "awaiting_approval"
}
```

## EnginesClient

Direct access to inference engines for custom model calls.

```typescript
// Complete with a specific engine
const result = await ctx.engines.complete({
  engine: "llamacpp",
  model: "gemma4",
  messages: [{ role: "user", content: "Hello" }],
});

// Embed with a specific engine
const vectors = await ctx.engines.embed({
  engine: "llamacpp-embed",
  texts: ["hello world"],
});
```

## RealtimeClient

WebSocket/RTC streaming for real-time interactions.

```typescript
// Broadcast to all connected clients
await ctx.realtime.broadcast("room:general", {
  type: "update",
  content: "Task completed",
});

// Subscribe to events
for await (const event of ctx.realtime.subscribe("room:general")) {
  console.log(event);
}
```

## How primitives are wired

Primitives are injected by Core's capability broker at runtime. The broker resolves each
abstract capability (`rag`, `memory`, `tts`, etc.) to a concrete provider app, spawns the
provider's sidecar, and injects the primitive client into the `RunnableContext`.

When running outside Core (e.g., `ryu dev`), only `gateway` is available. The other primitives
are `undefined`.

## Gateway enforcement

Every primitive that calls a model routes through the Gateway:

| Primitive | Gateway route | What it does |
|---|---|---|
| `gateway.chat` / `stream` | `/v1/chat/completions` | Full governance pipeline |
| `rag.embed` | `/v1/embeddings` | Embedding through Gateway |
| `engines.complete` | `/v1/chat/completions` | Direct engine via Gateway |

This means firewalls, budgets, rate limits, caching, and audit apply to every model call,
regardless of which primitive initiated it.
