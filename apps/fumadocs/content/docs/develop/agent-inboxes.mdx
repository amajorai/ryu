Agent Inboxes (Ryu Mail) is Ryu Cloud's hosted email service for agents, built on AWS SES. An agent
(or a user) gets a real, hosted inbox at `<local>@<domain>`: a unique address that stores every
email it receives and can send email over an API. It is the AgentMail-equivalent Ryu ships as its
own product.

<Callout type="info">
  Naming: the user-facing feature is **Agent Inboxes / Ryu Mail**, Ryu's own product. "AgentMail" is
  the third-party product this replicates, not a dependency Ryu resells.
</Callout>

## Where it lives

Inboxes, ownership, and send quotas are "what is allowed, measured, or paid for", so per the
Core-vs-Gateway placement rule this is **control plane**, not Core. The pieces:

| Piece | Path |
|---|---|
| Service package (SES send, S3 store, MIME parse, SNS verify) | `packages/mail/` (`@ryu/mail`) |
| REST API (`/api/mail/*`) | `packages/api/src/routers/mail.ts` |
| Plan quotas (single source of truth) | `packages/auth/src/lib/plans.ts` (`emailQuotaForPlan`) |
| Full setup + caveats | `docs/agentmail-ses.md` |

## The service package

`packages/mail/` (`@ryu/mail`) holds the building blocks, each lazy so importing the package never
touches AWS or throws on a missing env var:

- **Send.** `packages/mail/src/ses.ts` builds an SESv2 client lazily on the first send and exposes
  `sendEmail`. It throws `MailNotConfiguredError` when invoked without a configured domain, and
  rethrows SES errors verbatim so the caller can map them to a failed message plus an HTTP status.
- **MIME normalizer.** `packages/mail/src/parse.ts` turns raw RFC822 MIME into the stored message
  shape via postal-mime. `normalizeParsedEmail` is pure (no I/O), so it is unit-tested against a
  hand-built object; `parseRawMime` wraps the parser for the real inbound path. It groups replies by
  thread using the first reference, then the in-reply-to, then the message's own id.
- **Inbound (SNS) verification.** `packages/mail/src/inbound.ts` is the only publicly reachable
  surface, so it is fail-closed. `verifySnsSignature` cryptographically verifies each SNS message
  against its signing certificate, and the cert URL host is allow-listed to AWS SNS endpoints
  (`sns.<region>.amazonaws.com`), https-only. `confirmSubscription` auto-confirms a subscription
  only after the signature verifies and only when the `SubscribeURL` is likewise an allow-listed
  host. `extractSesInbound` reads the raw MIME from the inline SNS `content` when small, otherwise
  from the S3 object the receipt rule wrote.

## API

All endpoints live under `/api/mail` in `packages/api/src/routers/mail.ts`. Auth is either a Better
Auth **session** (the signed-in user plus their active org) or an `x-gateway-key` header (org-scoped,
the programmatic agent path). One `resolveScope` resolver handles both. When mail is not configured
every endpoint returns **503**, so it never crashes boot.

| Method and path | Purpose |
|---|---|
| `GET /mail/status` | `{ configured, domain }`, no auth |
| `GET /mail/inboxes` | List the caller's inboxes |
| `POST /mail/inboxes` | Create `{ name, localPart?, description?, webhookUrl? }`, plan-gated |
| `GET /mail/inboxes/:id` | A single inbox |
| `PATCH /mail/inboxes/:id` | Rename, toggle, or set the webhook |
| `DELETE /mail/inboxes/:id` | Delete the inbox and its messages |
| `GET /mail/inboxes/:id/messages` | List messages, `?direction=` and `?limit=` |
| `GET /mail/messages/:id` | One message, scoped through its inbox |
| `POST /mail/inboxes/:id/send` | Send `{ to, cc?, bcc?, subject, text?, html?, replyTo? }`, plan + quota gated |
| `POST /mail/inbound` | Public AWS SNS webhook for SES inbound |

The `x-gateway-key` path is what makes this an agent service: an agent presents its org's gateway key
and drives inbox CRUD, listing, and sending with no interactive session.

## Plan gating

Quotas are decided by `emailQuotaForPlan` in `packages/auth/src/lib/plans.ts`, the single source of
truth for the numbers. Inbox creation gates on a live entitlement; sending counts this scope's sent
outbound messages for the current calendar month from Mongo (indexed) rather than a per-send billing
round-trip.

| Plan | Inboxes | Sends per month |
|---|---|---|
| Pro | 3 | 1,000 |
| Max | 25 | 25,000 |
| Teams | 50 (org-wide) | 100,000 |
| Desktop license / free | off | off |

To change a number, edit one row in `packages/auth/src/lib/plans.ts`. The pricing page strings are
marketing copy; these are the enforced limits.

## Configuration

Every mail env var is optional. With no `RYU_MAIL_DOMAIN` set the feature is off and every endpoint
returns 503. The full setup, including the SES domain verification, the inbound MX record, the S3
bucket, and the SNS topic subscription, lives in `docs/agentmail-ses.md`.

```
RYU_MAIL_DOMAIN=mail.yourdomain.com   # the only var required to turn it on
AWS_SES_REGION=us-east-1              # falls back to STORAGE_REGION, then us-east-1
AWS_ACCESS_KEY_ID=...                 # falls back to STORAGE_ACCESS_KEY_ID, then the ambient chain
AWS_SECRET_ACCESS_KEY=...             # falls back to STORAGE_SECRET_ACCESS_KEY
RYU_MAIL_S3_BUCKET=ryu-mail-inbound   # raw MIME + attachments
RYU_MAIL_INBOUND_SECRET=...           # optional ?token= shared secret on the webhook
```

## Verification status

<Callout type="warn">
  The service package, plan wiring, and landing/pricing copy are shipped, but real AWS round-trips
  are not yet verified.

  - **Unit-tested:** the MIME normalizer against a raw `.eml` fixture
    (`packages/mail/src/parse.test.ts`), inbox address generation, and the plan email quota.
  - **Not verified here (needs live AWS):** a real SES send, real SNS to SES inbound delivery, and
    the subscription-confirmation round-trip all require a deployed, SES-verified domain plus the
    receipt rule and SNS topic wired up. A fresh SES account is sandbox-limited (it can only send to
    verified addresses) and inbound is region-restricted.
  - **No management UI yet.** This ships the control-plane service plus plan wiring and landing copy.
    A desktop or web inbox-management surface is a follow-on.
</Callout>
