---
title: "Build an agent just by chatting"
description: "Describe the agent you want in plain language and let the builder studio configure its model, tools, and persona for you."
---

You want a new agent, but you would rather describe it than fill in a dozen settings. This recipe
uses the agent builder studio: you tell a builder chat what you want in plain language, and it
configures the agent's model, tools, and persona for you. It is for anyone who wants a tailored agent
without learning every field first.

## What you'll use

- [Agents](/docs/using-ryu/user-guide/agents): the agent card model and the builder studio.
- [A tool the model can call](/docs/cookbook/a-tool-the-model-can-call): for when you want a custom tool, not a built-in one.

## Steps

<Steps>

<Step>

### Open the Agents page and start one

Each agent is a swappable "card": its own chat model, tools, memory, and persona. Open the **Agents**
page and click **+** to start a new one.

<TryInRyu page="agents" />

</Step>

<Step>

### Describe what you want in the builder chat

The agent edit page is a two-pane studio. The left pane is a builder chat that edits *this* agent;
the right pane is the organized config (model and provider, tools, connections, rules, instructions).

Tell the builder what you want in plain language:

> Make a formal research assistant that uses web search and answers with citations.

The builder applies the change for you. Under the hood it is offered Core's built-in `agent_builder`
tools (`get_agent`, `configure_agent`, `create_agent`); when you ask for a change, it calls
`configure_agent` to patch the persisted agent record, and the right-hand panel re-hydrates once the
turn settles. For a brand new agent, the first builder message mints a draft so there is something to
edit.

<Callout type="info">
The builder conversation runs as the flagship **Ryu** agent runtime with `persist: false`, so these
builder turns never enter your chat history. The built-in, locked `ryu` agent itself is edited
through its own config, not this builder.
</Callout>

</Step>

<Step>

### Refine and review

Keep iterating: "add the spaces memory", "switch the model to a local one", "make the tone
concise". Each request is one tool call against the agent record, and the right pane shows the
current state, so you can always see exactly what the builder changed.

</Step>

<Step>

### Chat with it

Save, and the new agent appears in the chat picker. Select it and your messages route through that
card's model, tools, and persona.

<TryInRyu page="chat" />

</Step>

</Steps>
