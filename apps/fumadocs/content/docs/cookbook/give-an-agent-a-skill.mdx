---
title: "Give an agent a new skill"
description: "Install an Agent Skill from skills.sh, enable it, and scope it to one agent's allowlist."
---

You want an agent to be good at a specific thing, code review, a writing style, a domain workflow,
without writing a prompt from scratch. This recipe installs an Agent Skill from the public catalog,
enables it, and scopes it to a single agent. It is for anyone who wants to teach an agent a reusable
capability in a couple of clicks.

## What you'll use

- [Skills](/docs/desktop/skills): browse and install Agent Skills from skills.sh.
- [Skills catalog](/docs/core/skills-catalog): the registry, activation set, and per-agent allowlist.

## Steps

<Steps>

<Step>

### Browse and install a skill

The **Skills** tab browses the public [skills.sh](https://skills.sh) directory. Find a skill and
install it; Ryu writes it in the universal Agent Skills layout (`~/.claude/skills/<slug>/SKILL.md`),
the same directory Claude Code and the skills CLI read, so an installed skill is shared across them.

<TryInRyu page="skills" />

Over the API the same flow is:

```http
GET  /api/skills/catalog
POST /api/skills/catalog/install
```

</Step>

<Step>

### Know the difference between installed and active

Installed is not the same as active. Because the shared skills directory can hold dozens of skills,
injecting every one would overflow a small model's context, so Ryu gates injection on an
**activation set**. A skill you install *through Ryu* is active automatically; skills discovered in
the shared directory are visible but inactive until you toggle them on. Flip the switch on the Skills
page, or over the API:

```http
POST /api/skills/activate
{ "id": "owner/repo/slug", "active": true }
```

</Step>

<Step>

### Scope it to one agent

By default an agent gets every enabled skill. To give *one* agent a skill (and not the others), set
that agent's **skill allowlist** on its edit page. An empty allowlist means all enabled skills; a
non-empty one is the intersection with the globally-enabled set, so it never re-activates a skill you
turned off globally.

The skill is injected on both chat planes, so the flagship `ryu` agent and other ACP agents both
receive it.

</Step>

<Step>

### Use it

Chat with the agent. With the skill active and allowlisted, its instructions are available to the
agent, either injected into the prompt or loaded on demand through the `skills` tool, depending on
your progressive-disclosure setting (Settings -> Memory -> "Load skills on demand").

</Step>

</Steps>

<Callout type="info">
Skills you install through Ryu are usable by Claude Code too, since they share the same
`~/.claude/skills` directory, and any skill those tools install shows up as installed in Ryu.
</Callout>

## Going further

- Author your own skill in code instead of installing one: [TypeScript SDK](/docs/develop/extensions/typescript-sdk) and the `defineSkill` factory.
- Decide which agent gets which tools, not just skills: [A tool the model can call](/docs/cookbook/a-tool-the-model-can-call).
- Keep big skills out of a small model's context: progressive disclosure in [Skills catalog](/docs/core/skills-catalog).
