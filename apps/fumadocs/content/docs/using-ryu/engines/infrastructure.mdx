Beyond chat, voice, and agent runtimes, Core manages a few infrastructure processes.

## Gateway

The Ryu Gateway runs as a managed sidecar (`apps/core/src/sidecar/gateway.rs`). It binds to `7981`
by default, and Core reaches it at `http://127.0.0.1:7981`. Core derives the Gateway's `--bind` from
`RYU_GATEWAY_URL` and sets provider env on spawn (including `LOCAL_LLM_URL` for the active local
engine). Set `RYU_GATEWAY_MANAGED=0` to run an external Gateway instead of letting Core spawn one.

The Gateway is on the default chat path. For everything it does, see [The Gateway](/docs/gateway).

## Headroom (context compression)

Headroom is an optional egress-compression proxy (default port `8787`). When enabled, the Gateway
posts request messages to headroom before the provider call to cut token usage, and it fails open if
headroom is unreachable.

- Enable with `RYU_HEADROOM_ENABLED=1`. Core then launches `headroom proxy`
  (`apps/core/src/sidecar/headroom.rs`) and sets the Gateway's compression env.
- The proxy URL is `RYU_HEADROOM_URL` (default `http://127.0.0.1:8787`).

See [Context compression](/docs/gateway/compression) for the full picture.

## Durable execution

Core can manage durable-execution engines for workflows.

- `restate` - the Restate server, the target for durable workflow steps and human-in-the-loop gates.
- `temporal` - an alternative workflow orchestrator.

<Callout type="info">
  Durable execution is a scaffolded spike today. The in-process, file-backed workflow engine
  (see [Workflows](/docs/core/workflows)) is what runs; the Restate and Temporal sidecars exist for
  the durable path that is still being built out.
</Callout>

## Tool sidecars

Core also manages tool processes such as `spider` (web scraping), `screenpipe`, `ghost` (desktop
automation), and `shadow` (capture and search). Ghost and Shadow are surfaced as built-in tool
servers in the [MCP registry](/docs/core/mcp-registry).
