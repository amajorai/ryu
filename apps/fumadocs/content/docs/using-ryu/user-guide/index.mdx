This page is a quick orientation to the Ryu desktop app: where things are and what each area
does.

## Layout

The desktop is organized into three parts:

- **Sidebar** (left) - your entry points and chat history. It holds **New chat**,
  **Search chats**, quick links to **Agents**, **Models**, and **Skills**, and the running
  list of your past conversations.
- **Main area** (center) - the active page, usually the chat view, but it switches to whatever
  page you open (Services, Gateway, Spaces, and so on).
- **Tabs** (top) - pages open as tabs so you can keep several surfaces open at once and switch
  between them.

## Navigation

Press **Ctrl+K** (or **Cmd+K**) to open the command palette and jump to any surface. The full
set of pages includes:

- **Chat** - talk to your agents.
- **Agents** - browse, create, and configure agents.
- **Services** - install and run agent runtimes (sidecars).
- **Gateway** - the control layer that governs model calls (routing, firewall, budgets, audit).
- **Engines** - the underlying agent engines and the active-engine swap.
- **Models** - browse and install local GGUF models from the Hugging Face catalog.
- **Skills** - browse and install Agent Skills from the skills.sh directory.
- **Spaces** - vector databases for retrieval-augmented generation (RAG).
- **Memory** - encrypted long-term memory.
- **Tools** - MCP tools and connections.
- **Workflows** - multi-step DAG workflows.
- **Automations** and **Delegation** - scheduled runs and sub-agent delegation.
- **Settings** - app, account, and appearance preferences.

<Callout type="info">
  Not every page is pinned to the sidebar. The sidebar keeps the most common actions one click
  away; the command palette is the fastest way to reach the rest.
</Callout>

## Split view

Tabs can also be shown side by side. A **split view** is a Zen-browser-style layout that places
two or more tabs in the main content area at once, as either **columns** (side by side) or
**rows** (stacked). Split membership lives on the tab itself (`tab.splitId` in
`apps/desktop/src/contexts/TabsContext.tsx`), so the split shows whenever the focused tab belongs
to it.

- Press **Ctrl+Alt+S** (or **Cmd+Alt+S**) to toggle a split view on the active tab
  (`apps/desktop/src/components/layout/TitleBar.tsx`).
- Tab context menus also create, drop a tab from, or dissolve a split.
- Drag the gutter between two panes to resize them. Panes share the cross-axis fully and divide
  the main axis by fraction, clamped so neither pane can collapse
  (`apps/desktop/src/components/layout/SplitView.tsx`).

<Callout type="info">
  Panes are absolute-positioned rather than reparented, so switching focus between panes never
  reloads a page. The frosted titlebar overlays the top of the content area, so a column pane or
  the top stacked row pads its content down to clear it.
</Callout>

## Background customization

You can give the sidebar and the page area custom backgrounds from **Settings → Appearance**
(`apps/desktop/src/components/settings/BackgroundCustomizationSettings.tsx`). Each of the two
surfaces, **sidebar** and **page**, takes an independent background built from up to three layers,
composited top to bottom: an overlay tint, a custom image, and a linear gradient, with the solid
theme color showing through any transparency.

- **Gradient** - two hex colors and an angle.
- **Image** - any picked image, with a **fit** of cover, contain, stretch, center, or tile, plus a
  scale.
- **Overlay** - a tint layered over the image for legibility.

The small config (gradient, fit, scale, overlay, toggles) is stored in `localStorage`, applied as
CSS variables; the image bytes live in IndexedDB as a blob, so there is no practical size limit.
Changes apply live and sync across open windows.

<Callout type="info">
  Backgrounds are off by default, so a fresh install uses the plain theme. The Calendar and
  Timeline pages draw their own surface, so they occlude a page background image.
</Callout>

## The Island companion

Ryu ships a separate **Island** companion: a frameless, dynamic-island-style overlay that
floats above your other windows on Windows and macOS. It monitors on-screen context, surfaces
proactive suggestions from a local model, and offers a mini chat onto Ryu Core, all behind a
per-capability consent gate that you control. It is opt-in: nothing is captured until you
grant permission.
