title: "Configuration"
description: "Environment variables for the Ryu Core MCP server and how to point an MCP host at a remote node over Tailscale instead of a local Core."
---

The MCP server is configured entirely through environment variables. There is no config file of its own; it inherits everything it needs from the environment the host launches it in.

## Environment variables

- `RYU_CORE_URL` - the base URL of the node to drive, for example `http://127.0.0.1:7980` for a local Core or a Tailscale URL for a remote one. Defaults to `http://127.0.0.1:7980`.
- `RYU_CORE_TOKEN` - the node-admittance token (`RYU_TOKEN`) the node expects on `Authorization: Bearer`. The bridge sends it on every request to Core. Optional - a local loopback node enforces none.
- `RYU_AUTH_URL` - the control-plane (Better Auth) base URL, used only by the `login` / `logout` / `whoami` subcommands and the `ryu_whoami` tool. Defaults to `http://localhost:3000`.

Point `RYU_CORE_URL` and `RYU_CORE_TOKEN` at any reachable Core and the same host now drives that node.

## Pointing at a remote node

Because the surface and the brain are already separate in Ryu, driving a remote node is just a different `RYU_CORE_URL`. Put the node on your tailnet and give the bridge its Tailscale URL and the matching token:

```json
{
  "mcpServers": {
    "ryu-remote": {
      "command": "bun",
      "args": ["run", "apps/mcp"],
      "env": {
        "RYU_CORE_URL": "https://my-node.tailnet.ts.net:7980",
        "RYU_CORE_TOKEN": "remote-node-token"
      }
    }
  }
}
```

Nothing else changes. The host sees the same tools; they now act on the remote node, and the remote node's Gateway governs the calls.

<Callout type="info">
  You can register more than one Ryu server in the same host: a `ryu` pointed at your local Core and a `ryu-remote` pointed at a node on your tailnet, each with its own token. Name them distinctly so the host's tool list stays readable.
</Callout>

## Multiple nodes, one host

Running several entries side by side is the supported way to switch nodes. The host shows each server's tools under its own name, so `ryu_ask` on `ryu` hits your laptop and `ryu_ask` on `ryu-remote` hits the tailnet node. Keep tokens per node so revoking one does not affect the others. See [security](/docs/mcp/security).
