Claude Code and Codex are both MCP hosts, so the same `apps/mcp` bridge that connects Claude Desktop can put a running Ryu Core inside your coding agent. Ryu ships this as an installable plugin for each agent (`apps/plugins/claude-code` and `apps/plugins/codex`), so you do not hand-edit a host config file. Enable the plugin and the agent gains Ryu's tools plus, in Claude Code, a set of `/ryu:*` slash commands.

The plugin wraps three things Ryu already ships, so there is no new backend: the MCP bridge at `apps/mcp`, two skills (`setup-ryu`, `ryu-tools`), and the command flows. See [MCP Server](/docs/mcp/index) for how the bridge itself works and [Tools](/docs/mcp/tools) for the full tool list.

<Callout type="warn">
  The bridge launches from this monorepo (`apps/mcp/src/index.ts`) and depends on the `@ryuhq/core-client` workspace package, so run the coding agent **from the Ryu repo root**. To use the plugin from another project, point the launch command at an absolute path to your checkout (`claude-code/.mcp.json` or `codex/.mcp.json` in `apps/plugins`). Built 2026-07-08.
</Callout>

## What the plugin gives you

Once enabled, the agent can drive your node with the ~21 `ryu_*` tools the bridge exposes - health and system status, model search and activation, agent and team listing, workflow runs, RAG search across memory and Spaces, a one-shot `ryu_ask`, and `ryu_call_mcp_tool` to reach any MCP server Ryu has registered. ## Prerequisites

- **Bun** on PATH - the bridge runs under `bun`.
- A reachable Ryu Core node. A local desktop or headless Core listens on `http://127.0.0.1:7980` (see [Quickstart](/docs/mcp/quickstart) for starting one).

## Install into Claude Code

The marketplace manifest lives at `apps/plugins/.claude-plugin/marketplace.json`. From the Ryu repo root:

```sh
claude plugin marketplace add ./apps/plugins
claude plugin install ryu@ryu
```

On enable, Claude Code prompts for the plugin's two `userConfig` options (`claude-code/.claude-plugin/manifest.json`):

- `ryu_core_url` - base URL of your node, default `http://127.0.0.1:7980`.
- `ryu_core_token` - optional node-admittance bearer (marked sensitive). Leave blank for a local loopback node.

Those values flow into the plugin's `.mcp.json`, which launches the bridge as the `ryu` MCP server and passes them through as the `RYU_CORE_URL` and `RYU_CORE_TOKEN` environment variables.

```json
{
  "mcpServers": {
    "ryu": {
      "command": "bun",
      "args": ["run", "${CLAUDE_PROJECT_DIR}/apps/mcp/src/index.ts"],
      "env": {
        "RYU_CORE_URL": "${user_config.ryu_core_url}",
        "RYU_CORE_TOKEN": "${user_config.ryu_core_token}"
      }
    }
  }
}
```

The tools appear the moment the plugin is enabled. In a session started from the repo root, confirm the wiring:

```
/ryu:status
```

## Slash commands (Claude Code)

The Claude Code plugin ships five slash commands (`apps/plugins/claude-code/commands/`). Each is a short flow over the `ryu_*` tools:

| Command | What it does | Tools it drives |
|---|---|---|
| `/ryu:status` | Report node health, active engine and running state, gateway reachability, active model, and a hardware headline. Stops if Core is unreachable. | `ryu_health`, `ryu_system_status`, `ryu_get_active_model`, `ryu_system_info` |
| `/ryu:models [query]` | Search the model catalog and optionally activate an installed model, confirming the switch. | `ryu_search_models`, `ryu_get_active_model`, `ryu_set_active_model` |
| `/ryu:ask <question>` | Ask your node one synchronous question, routed to its side-question model (no tools, no agent run). | `ryu_ask` |
| `/ryu:workflow [id]` | List the workflows on your node and run one by id, gathering inputs first and reporting the run state. | `ryu_list_workflows`, `ryu_run_workflow` |
| `/ryu:retrieval <query>` | Unified RAG search across memory and every Space, or scope to a single Space. | `ryu_search_retrieval`, `ryu_list_spaces`, `ryu_search_space` |

`/ryu:models`, `/ryu:workflow`, and `/ryu:ask` change state or consume model budget - see the write-tool caution in [security](/docs/mcp/security).

## Install into Codex

The Codex marketplace manifest lives at the repo-root `.agents/plugins/marketplace.json`. From the Ryu repo root:

```sh
codex plugin marketplace add ./.agents/plugins
codex plugin install ryu
```

Codex has no slash-command surface, so the Codex plugin (`codex/.codex-plugin/manifest.json`) carries the same flows through its `setup-ryu` and `ryu-tools` skills plus a `defaultPrompt` list of starter suggestions (check status, search and activate a model, ask the node, list and run a workflow, search memory and Spaces). Its `.mcp.json` launches the same bridge:

```json
{
  "mcp_servers": {
    "ryu": {
      "command": "bun",
      "args": ["run", "apps/mcp/src/index.ts"],
      "env": {
        "RYU_CORE_URL": "http://127.0.0.1:7980",
        "RYU_CORE_TOKEN": ""
      }
    }
  }
}
```

Unlike the Claude Code manifest, the Codex `.mcp.json` sets the target directly, so edit `RYU_CORE_URL` / `RYU_CORE_TOKEN` there to point at a non-default node. Toggle the plugin off any time with `enabled = false` under it in `~/.codex/config.toml`.
