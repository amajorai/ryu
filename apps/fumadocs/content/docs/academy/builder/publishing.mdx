When your app is ready, you publish it to the **marketplace** so others can find and install it. Publishing is governed by the Gateway, which means the platform checks and signs your app before it can go out.

## Publish from the SDK

The SDK exposes a `ryu publish` target:

```bash
ryu publish
```

That single command kicks off the governed flow.

## What the Gateway does

Publishing routes through the Gateway, which enforces three things:

- **Grant validation** - it confirms the app's requested grants, and it is fail-closed (no valid grant, no publish).
- **ed25519 signing** - the Gateway signs the manifest.
- **Verification** - the signed manifest is verified against a pinned public key, not a key carried in the manifest itself.

<Callout type="warn">
  Because grant validation is fail-closed, anything unverified is rejected by default. The pinned public key means a tampered manifest cannot vouch for itself.
</Callout>

## The federated marketplace

<TryInRyu page="marketplace" />

The marketplace is federated. Every catalog (models, skills, MCP, and plugins) routes through one catalog-source seam, with built-in sources plus user-added sources and a persisted active selector.

A key rule: sources return **descriptors only**, while Core keeps the privileged install. A source can describe what is available, but it never gets to run the install step itself.

There is also a Ryu-hosted first-party marketplace. When you publish there, your app's status moves from **pending** to **live** before it is broadly listed.

## Safe fetching

Untrusted fetches are SSRF-guarded: https-only, IP-pinned, and size-capped. A malicious source cannot redirect Core to an internal address or hand it an oversized payload.
