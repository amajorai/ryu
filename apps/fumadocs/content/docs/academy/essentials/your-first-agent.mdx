Ryu installs exactly one agent by default, named **Ryu**. It is the flagship demo of the whole idea,
so it is worth understanding before you chat with it.

## What is an agent in Ryu?

Think of an agent as a **trading card**. It has independently swappable slots:

- a **chat model** that generates text
- optional **voice** and **image** models
- **tools and MCP servers** it is allowed to call
- **memory** and **Spaces** for what it can recall
- a **persona** and instructions
- a slice of **Gateway policy** that governs its calls

No two cards have to be alike. You can build an agent for coding with one model and toolset, and a
different agent for support with another, and switch between them per chat.

## The default Ryu agent

The default **Ryu** agent is Pi with the Gateway on top. Pi is a capable agent engine. The Gateway
is Ryu's control layer. Putting the Gateway in front of Pi is the headline demo of "the car around
the engine" - the same engine, now governed, routed, and measured.

The Ryu agent is self-sufficient. It manages its own engine binary, so it works without any extra
setup.

## Where to find agents

<TryInRyu page="agents" />

Open the **Agents** page from the sidebar or the command palette. You will see the installed set,
which starts as just the Ryu agent. From here you can later:

- browse the **agents catalog** and add other agents such as Claude Code, Codex, Gemini CLI, or bare
  Pi, if you have their tools installed
- create your own agent on a single page
- edit an agent's slots

You will do all of that in the [Operating Ryu](/docs/academy/operator) course. For now you only need
one agent, and you already have it.

<Callout type="info">
  The Ryu agent is a locked built-in. You can still tune its model and provider through a dedicated
  config panel, which you will see in the next course.
</Callout>
