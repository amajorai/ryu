Ryu is designed to work the moment it is installed, with no API key and no setup. This lesson walks
through what happens the first time you launch the app.

## Install the desktop app

Download the Ryu desktop app for your platform and install it like any other application. The
desktop app bundles **Ryu Core**, the local backend that does the real work, so you do not install
anything separately.

When the app starts for the first time, it runs **onboarding**. This is where Ryu sets up your local
stack.

## What onboarding downloads

Onboarding installs a complete local stack so Ryu is private and fully functional offline. Each step
is best-effort, so if one fails it warns you and never blocks chat.

- **A local chat engine and model** - llama.cpp plus a default Gemma GGUF model that runs on most
  machines. This is what answers your messages out of the box.
- **An embedding model** - a local GGUF used for retrieval and memory, served by a dedicated
  embeddings instance.
- **A voice engine** - whisper.cpp plus a base speech model for speech to text.
- **A speech model** - the parakeet ONNX model for an alternative transcription path.

Everything above runs locally. No data leaves your machine unless you later choose a cloud model or
connect an external service.

<Callout type="info">
  Onboarding detects which agent command-line tools you already have installed and offers to add
  the matching agents. Only the default **Ryu** agent is installed automatically. Everything else is
  opt-in.
</Callout>

## Sign in (optional)

You can sign in to sync conversations, manage billing, and use cloud features. Signing in is not
required to chat. The local stack works whether or not you have an account.

## Where things live

Two pieces matter for the rest of this course.

- **Sidebar** on the left holds New chat, search, and quick links to Agents, Models, and Skills.
- **Command palette** opens with **Ctrl+K** or **Cmd+K** and jumps you to any page in the app.

If onboarding finished without errors, you already have a working agent. Let us go meet it.
