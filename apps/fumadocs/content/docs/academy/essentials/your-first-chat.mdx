Time to actually talk to your agent. This lesson gets you to a real conversation and shows you the
moving parts you will use every day.

## Start a conversation

<TryInRyu page="chat" />

1. Click **New chat** in the sidebar.
2. Type a message and send it.

Your message goes to Ryu Core, which routes it to the default **Ryu** agent. The agent generates a
reply using your local model and streams it back token by token. That is the full loop, running
entirely on your machine.

## Watch a tool get used

A plain reply is just text generation. The interesting part is when an agent **uses a tool**. Ask it
to do something that needs one, for example to fetch a page or look something up.

Under the hood the agent runs a loop - it decides it needs a tool, calls the tool, reads the result,
and continues. You see the tool call and its result inline in the conversation. This is the
chat-to-agent-to-tool-to-result loop, and it is what makes an agent more than a chatbot.

<Callout type="info">
  Which tools an agent may call is governed, not unlimited. The agent can only reach tools on its
  allowlist, and every tool call is measured. You will learn to manage this in later courses.
</Callout>

## Useful controls

A few things you will reach for right away.

- **Stop** halts a running response.
- **Voice input** - the microphone button captures speech, transcribes it locally, and drops the
  text into the composer.
- **Branch** - from a message you can fork the conversation into a new chat that copies the history
  up to that point. Your conversations stay linear, and the branch opens in a new tab.
- **Sessions** - a read-only view of the runs that make up a conversation.

## What just happened

You ran a request all the way through the stack - the desktop UI sent it to Core, Core routed it to
an agent, the agent generated and optionally called a tool, and the result streamed back and was
saved. That round trip is the heartbeat of everything Ryu does.
