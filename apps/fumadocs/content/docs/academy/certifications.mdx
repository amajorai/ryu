Certifications are a way to prove you can actually do something with Ryu, not just that you read
about it. There are four levels, each mapped to one of the Academy tracks. Work through the track,
make sure you can do everything in the checklist, and you are ready.

<Callout type="info">
  The Academy is the preparation. Each certification below lists the track that prepares you for it
  and the skills you should be able to demonstrate.
</Callout>

## Certified User

**Prepares with** [Ryu Essentials](/docs/academy/essentials).

For everyone who uses Ryu day to day. You can be productive without writing code.

You should be able to:

- Explain what Ryu is and why it sits above any engine
- Install Ryu and complete first-run onboarding
- Start a chat, use voice input, and branch a conversation
- Recognize a tool call in a conversation and explain the agent loop

## Certified Operator

**Prepares with** [Operating Ryu](/docs/academy/operator).

For power users who set Ryu up for themselves or a team.

You should be able to:

- Create and edit an agent, treating each slot as swappable
- Browse and install models from the catalog and switch the active model
- Install and activate skills, and understand installed versus active
- Connect tools and reason about an agent's allowlist
- Use Spaces and memory for retrieval, and set up an automation or monitor

## Certified Gateway Admin

**Prepares with** [Governing Ryu](/docs/academy/governing).

For the person who owns policy. This is the moat layer.

You should be able to:

- Explain the Core versus Gateway split and why Core never enforces policy inline
- Configure model routing, including smart classifier-driven routing
- Set up guardrails such as the firewall and PII or DLP controls
- Apply budgets and manage keys with a BYOK vault
- Run evals and read the audit trail

## Certified Builder

**Prepares with** [Building on Ryu](/docs/academy/builder).

For developers who extend the platform.

You should be able to:

- Explain the Runnable contract and how Agent, Workflow, Tool, and Skill unify under it
- Scaffold a project with the SDK and define a Runnable
- Package an app with a manifest and understand install, enable, and disable
- Publish to the marketplace and explain how publishing is governed by the Gateway

## How to prepare

1. Pick the track that matches the level you want.
2. Work through every lesson, doing the steps in the real app, not just reading.
3. Run each lesson's knowledge check honestly.
4. Use the checklist above as your final self-assessment.

<Callout type="warn">
  These checklists are living documents. Ryu ships quickly, so when a track gains a lesson, the
  matching checklist gains a skill. Always prepare against the current track.
</Callout>
