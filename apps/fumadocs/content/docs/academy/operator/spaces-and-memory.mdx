Two slots on an agent card give it knowledge beyond the conversation in front of it. Spaces hold documents you can retrieve from, and memory holds things the agent should remember over time. Both are per-agent, so different cards can have different knowledge.

## Spaces

<TryInRyu page="spaces" />

A Space is a vector database for retrieval-augmented generation (RAG). Open the **Spaces** page to create a Space and add documents to it.

What makes Ryu's Spaces private is where the embedding happens. Embeddings are generated by a local embedding model (a nomic GGUF served on your own machine), so the contents of your documents do not leave your device to be indexed.

You can also edit markdown pages directly inside a Space. When you save, the page re-embeds automatically, so retrieval always reflects the latest text.

Under the hood, retrieval works by cosine similarity over the stored embeddings - the agent finds the chunks closest in meaning to the question and pulls them in.

<Callout type="info">
  Because embeddings are local, you can add sensitive documents to a Space without sending them to an external indexing service.
</Callout>

## Memory

Memory is encrypted long-term storage the agent can recall from later. Where a Space is a body of documents you curate, memory is the agent's own running recollection.

It is encrypted, and like Spaces it is a per-agent slot. One agent's memory is not another's.

## Per-agent by design

Keeping both slots per-agent is what lets you build sharply different cards:

- A research agent with a large documents Space and rich memory.
- A throwaway drafting agent with neither.

Fill the slots to match the job, not the other way around.

<Callout type="warn">
  Retrieval quality depends on what is in the Space. If answers seem to miss obvious facts, check that the relevant document was added and has re-embedded after edits.
</Callout>
