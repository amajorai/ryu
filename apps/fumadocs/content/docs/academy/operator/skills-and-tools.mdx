Skills and tools are the two ways you extend what an agent can actually do. They are different things, so this lesson covers each in turn.

## Skills

<TryInRyu page="skills" />

Skills are reusable Agent Skills in the standard `SKILL.md` format. The **Skills** page browses the public skills.sh directory and installs skills into the universal `~/.claude/skills` location.

That shared location is the point - a skill you install through Ryu also works in Claude Code, and a skill Claude Code installed shows up in Ryu. One install, usable across tools.

### Installed is not the same as active

This distinction matters more than it sounds. Skills you install through Ryu are active right away. Skills that Ryu discovers in the shared folder are visible and counted as installed, but they stay inactive until you toggle them on.

The reason is context - injecting every skill body into a local model at once would overflow its context window, so Ryu only injects the ones you have chosen to activate.

<Callout type="info">
  If a skill is installed but does not seem to take effect, check that it is toggled active on the Skills page.
</Callout>

## Tools

Tools come from MCP servers and a unified tool catalog. The **Tools** page manages your MCP tools and connections.

Two rules govern how an agent uses them:

- An agent can only call tools that are on its allowlist.
- Every tool call is governed and measured by the Gateway.

This means tool access is per-agent. Adding a tool to the catalog does not give every agent access to it. You grant it on the agent's allowlist.

### Composio and built-ins

Composio provides a large set of third-party tool integrations, gated per agent so each card only reaches what you allow. Ryu also ships built-in tools, including desktop notifications and channel sends.

<Callout type="warn">
  An allowlist is a security boundary, not a convenience setting. Only allow the tools an agent genuinely needs.
</Callout>
