Agents are commoditized. The control layer around them is not. This course is for the person who owns policy: the Gateway is where you decide what an agent is allowed to do, what data is safe to send, which model serves a request, what it costs, and how you prove what happened.

The good news is that the Gateway is already on the default chat path. This course is not about turning it on, it is about learning its surfaces and using them well.

## Who this is for

- Platform and developer teams who run agents for others and need a single control point.
- Anyone responsible for security, spend, model choice, or compliance across a team or organization.
- The owner who wants the moat layer (the part a team adopts and an enterprise buys) to actually be configured.

## What you will learn

- The single most important design rule: Core decides what runs, the Gateway decides what is allowed.
- How model routing works, including smart classifier-driven routing from plain-language rules.
- How the firewall and PII/DLP guardrails protect requests and egress.
- How budgets cap spend, how BYOK stores keys, and how BYOA points any agent at the Gateway.
- How evals measure output quality and audit proves what happened.
