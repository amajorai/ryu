Ryu is built as open core plus managed cloud, the same shape as Vercel and Supabase. The Rust Core and Gateway are open-source and self-hostable; the desktop, web, and backend are closed. This track is for teams who want to run Ryu beyond a single laptop: self-host it, put agents into chat platforms, and spread work across machines.

Ryu Cloud is the "we audit, build, deploy, and host" offer, aimed at small and medium businesses in Singapore and Southeast Asia. Part of it is a product you run, and part of it is a service A Major provides. This course is honest about which is which.

## Who this is for

- Teams and businesses, not just solo users
- Anyone deciding whether to self-host or use managed Ryu Cloud
- People who want agents reachable from Telegram, Slack, WhatsApp, or Discord
- Operators running Ryu across more than one machine

## What you will learn

- What actually gets deployed and how a hosted node stays governed
- How channel bots turn a chat platform into another surface onto the same agent
- How multiple nodes, mesh, and sync work together
- Where managed Cloud is a feature you can use today versus a service offering
