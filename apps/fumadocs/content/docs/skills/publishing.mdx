Skills live in one well-known place, and Ryu reads them the same way Claude Code and the skills CLI do. This page covers where a skill goes, how an agent finds it, and how it becomes active.

## The universal location

Skills load from the universal Agent Skills directory:

```bash
~/.claude/skills/<id>/SKILL.md
```

This is overridable with `RYU_SKILLS_DIR`, and the legacy flat `<id>.md` layout still loads for back-compat. Because the path is the same one Claude Code reads, a skill you install for Ryu is available to those tools too, and the reverse holds. Publishing a skill is, at its simplest, getting its `SKILL.md` into this directory on the node.

## Installed versus active

Installing a skill is not the same as activating it. A skill in the directory is installed. Whether it is active for a given turn is a separate decision:

- The `enabled` front-matter flag gates the skill globally. An `enabled: false` skill is installed but never selected.
- A per-agent allowlist gates which installed skills an agent may use, so two agents on the same node can see different skill sets.
- Under progressive disclosure, an active skill still only loads its full body when the model decides it is relevant.

This split is what keeps a large library cheap. You can install many skills and let selection and disclosure surface only the ones a turn needs.

## How an agent discovers and loads a skill

On the ACP plane, an agent works with skills through tools:

- `skills__search` lists the installed skills and their descriptions, so the model can find one by intent.
- `skills__load` pulls a specific skill's full body into context on demand.

Core injects each active skill's name and description up front, then lets the model load bodies as needed. When the model loads a skill, Core injects its instructions into the assembled request body before forwarding to the Gateway, so the turn demonstrably changes and the Gateway attributes the call to the skill through the `x-ryu-skill-ids` header.

<Callout type="warn">
  Skills are instructions, not sandboxes. A skill can grant tools through `allowed-tools`, but the Gateway still enforces every grant. Review a skill's body and its tool list before installing one from a source you do not control.
</Callout>

## Shipping with the collection

To add a skill to Ryu's maintained set, contribute it to the `apps/skills` collection so it ships alongside [setup-ryu](/docs/skills/setup-ryu) and the rest of the [catalog](/docs/skills/catalog). The collection is just `SKILL.md` files laid out by id, the same shape an agent reads at runtime.
