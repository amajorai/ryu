The `apps/skills` directory is Ryu's maintained collection of agent skills. Each one is a `SKILL.md` instruction pack you can install to the universal skills directory and activate per agent. The collection is led by the [setup-ryu](/docs/skills/setup-ryu) flagship and rounds out with packs for the parts of Ryu an agent most often needs to reason about.

<Callout type="info">
  The catalog grows over time. Treat this page as the curated set Ryu ships rather than an exhaustive registry. Run `skills__search` from an agent to see what is installed on your node right now.
</Callout>

## Shipped skills

| Skill | What it teaches |
|---|---|
| **setup-ryu** | The flagship. Teaches an agent how Ryu is structured so it can install, configure, and operate a node accurately. See [setup-ryu](/docs/skills/setup-ryu). |
| **ryu-local-model** | How to browse, download, and run local GGUF/MLX models through the model catalog and llama.cpp sidecar. |
| **ryu-mcp** | How to register, configure, and use MCP servers through Core's MCP registry, and which tools read versus write. |
| **ryu-author-skill** | How to author a new Ryu agent skill: the SKILL.md format, progressive disclosure, and how to test and publish. |
| **ryu-build-agent** | How to build a custom agent in code using the Ryu TypeScript SDK, from `defineAgent` to install. |

## How they relate

`setup-ryu` is the entry point. The rest are focused packs you load when an agent will work in that area. Under progressive disclosure you can keep all of them installed and let the model load only the ones a turn needs, so a large catalog costs almost nothing until a skill is actually relevant. See [Skills](/docs/skills) for how disclosure works.

To write your own and add it to a collection, see [authoring](/docs/skills/authoring) and [publishing](/docs/skills/publishing).
