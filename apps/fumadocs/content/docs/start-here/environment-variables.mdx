This page is the canonical inventory of environment variables in the Ryu monorepo. It is assembled from `packages/env` Zod schemas, per-app `.env.example` files, and code scans across Rust and TypeScript apps.

<Callout type="info">
  **Legend.** *Required* means the app fails to start or a feature is unusable without the var in that deployment mode. *Optional* means a built-in default exists or the feature is opt-in. *Required\** on `.env.example` rows means the example shows it uncommented for local dev - check the schema for production rules.
</Callout>

## How configuration is organized

| Layer | Mechanism | Apps |
|---|---|---|
| Typed schemas | `@ryu/env` (`@t3-oss/env` + Zod) | `apps/server`, `apps/web`, `apps/native`, `packages/{auth,db,api,email,mail}` |
| Env-only (no schema) | `process.env` / `import.meta.env` / `std::env` | `apps/core`, `apps/gateway`, desktop/webapp/extension (Vite), `cloud-bot`, `mcp`, `tui`, `island` |
| TOML + env overrides | `gateway.toml` + `GATEWAY_*` env | `apps/gateway` |
| User preferences | Raycast extension prefs (not env) | `apps/raycast` |

Copy the matching `.env.example` to `.env` (or `.env.local`) in each app directory. Core and Gateway have **no config file** beyond env vars (Gateway also reads `gateway.toml`).

## Gap analysis

### Vars in `packages/env/web` schema but also read outside it

`NEXT_PUBLIC_SERVER_URL` is consumed by `packages/email` server-side (should be in server schema or a shared export).

### Vars in `.env.example` but not in `packages/env/server` schema

`apps/server/.env.example` documents `DUB_PARTNERS_*`, `POLAR_BENEFIT_DESKTOP_LICENSE`, `POLAR_METER_MANAGED_INFERENCE`, `POLAR_PRODUCT_CLOUD_CLOUD*`, `RYU_MARKETPLACE_API_URL` — legacy or comment-only.

### Remaining gaps

- **No Vite schema:** `VITE_*` vars for desktop, webapp, and extension are ad hoc with no Zod validation.
- **Core vars in code not in `.env.example`:** e.g. `RYU_ADVISOR_MODEL`, `RYU_DOUBLE_CHECK_MODEL`, `GEMINI_API_KEY`, `GROQ_API_KEY`, and other Pi provider keys in `apps/core/src/pi_config/mod.rs`.
- **turbo.json** lists `.env*` as build inputs only - no env passthrough whitelist.
