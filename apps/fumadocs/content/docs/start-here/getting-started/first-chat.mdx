When Ryu opens, you land on the chat screen with **Ryu** already selected as the agent. Ryu is
the flagship built-in: it is [Pi](/docs/using-ryu/engines/agent-runtimes) with the
[Gateway](/docs/gateway) running on top of it, and it is the only agent installed by default
(`DEFAULT_AGENT_ID` is `"ryu"`, set in `apps/core/src/sidecar/adapters/acp.rs`). It manages its
own Pi binary and runs locally, so there is no API key, no terminal, and no setup before your
first message.

## Send your first message

<Steps>

### Open a fresh conversation

Click **New chat** in the sidebar, or just start typing on the empty chat screen. The active
agent shown at the top of the composer is **Ryu**.

### Type a message

Enter your message in the composer at the bottom of the screen. Try something simple to confirm
the local engine is answering, for example `Summarize what you can do.`

### Send it

Press **Enter** to send. Use **Shift+Enter** to add a new line without sending.

</Steps>

The reply streams in token by token. Behind the scenes, Ryu routes the turn through the Gateway
to its managed Pi engine and back. Your conversation is saved automatically and appears in the
chat history in the sidebar, so you can return to it later. The mechanics of how that history is
stored live in [Conversations and sessions](/docs/core/conversations-sessions).

<Callout type="info">
  Ryu works fully local and private on install: the default engine is llama.cpp serving Gemma,
  downloaded during onboarding. See
  [Batteries-included defaults](/docs/start-here/architecture/batteries-included) for the full
  default stack.
</Callout>

## Find the command palette

Press **Cmd+K** (macOS) or **Ctrl+K** (Windows and Linux) anywhere in the app to open the
command palette (`apps/desktop/src/components/layout/CommandPalette.tsx`). From there you can jump
straight to Chat, Agents, Models, Skills, Spaces, Memory, Tools, Workflows, and the rest of the
app without hunting through the UI. It is the fastest way to move around Ryu.

## Add other agents

Every agent is a self-contained card: its own model, tools, memory, and persona. The agent picker
in the sidebar lists only the agents that are **installed** (backed by `GET /api/agents`), and on
a fresh install that is just Ryu.

To add a coding agent such as Claude Code, Codex, or Gemini CLI, install its runtime first. Each
runs as a subprocess managed by Ryu Core and supports a full tool loop, calling a tool, reading
the result, and continuing across turns within a single reply. That is what lets them read files,
run commands, and edit code rather than only chat.
