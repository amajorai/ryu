title: "Batteries Included"
description: "What ships on first install with zero API keys: a local chat model, embeddings, voice, default skills and tools"
---

On install, with no API key and no setup, Ryu works immediately and stays fully local and
private. One Core routine, `install_local_stack` in `apps/core/src/sidecar/onboarding.rs`, is the
single owner of every default model download. The engines only ever *serve* these files; they
never download them.

Every default below is a swappable default, never a lock. Each layer routes through one
registry or config, so you can replace the chat model, the embedder, the voice engine, or the
agent without touching anything else. See [Why Ryu](/docs/start-here/architecture/why-ryu) for
the "nothing hardcoded" principle this enforces.

<Callout type="warn">
  Each download step is non-fatal and independent. A failure (offline, no prebuilt binary on a
  given OS) is recorded as a warning and never blocks chat. If `llama.cpp` fails to install, the
  chat and embedding GGUF downloads are skipped, RAG falls back to a local hashing embedder, and
  chat falls back to a configured remote route.
</Callout>

## What downloads on first run

These are fetched in order by `install_local_stack`. Each is checksum-verified through the shared
download pipeline and marked installed so the matching sidecar becomes eligible to start.

| Step | What | Sidecar / engine | Purpose |
|---|---|---|---|
| 1 | `llama.cpp` (`llama-server` binary) | `llamacpp` | Local chat inference runtime |
| 1 | Same binary registered for embeddings | `llamacpp-embed` (:8081, auto-started) | Serves the embedding model |
| 2 | Default chat GGUF (Gemma 4) + matching vision adapter (`mmproj`) | `llamacpp` | The default chat model, multimodal out of the box |
| 3 | `nomic-embed-text-v1.5` GGUF | `llamacpp-embed` | Semantic embeddings for RAG, memory, search |
| 4 | `whisper.cpp` server + `ggml-base.en` model | `whispercpp` | Speech-to-text (STT) |
| 5 | parakeet v3 ONNX speech model | `parakeet` | Alternative STT engine bundle |
| 6 | OuteTTS + WavTokenizer GGUFs (`llama-tts` binary) | `outetts` | Text-to-speech (TTS) |

<Callout type="warn">
  parakeet *inference* is gated behind the off-by-default `voice-parakeet` build feature. The
  model download in step 5 is independent, so the bits are present, but transcription returns a
  "feature not built" error unless Core was compiled with that feature. whisper.cpp is the STT
  engine that works out of the box. See [Voice engines](/docs/desktop/engines/voice-engines).
</Callout>

The vision adapter in step 2 is bound to the chat model's stem by the `<stem>.mmproj.gguf`
convention, so a multimodal default like Gemma accepts image input on a fresh install. A
text-only default, or a failure to fetch the adapter, leaves the model chatting as text. Engines
that are not in the startup order (whisper, parakeet, OuteTTS) install ready but stay opt-in to
*run*: installing only makes them eligible to start.

## Defaults that work without a download

These ship as config or registry defaults rather than downloaded artifacts. They are active on
install with no setup.

| Default | What it is | Where to swap |
|---|---|---|
| Default agent "Ryu" | Pi with the Gateway on top, the only agent installed by default; `DEFAULT_AGENT_ID` is `"ryu"` | [Agents](/docs/desktop/user-guide/agents) |
| Reranker | BAAI `bge-reranker`, the default RAG reranker | [Spaces and RAG](/docs/core/spaces-rag) |
| RAG strategy | Vector retrieval over sqlite-vec, with GraphRAG planned | [Spaces and RAG](/docs/core/spaces-rag) |
| `agentbrowser` | Web browsing tool app | [Tools and connections](/docs/desktop/user-guide/tools-and-connections) |
| `spider` | Web scraping tool app | [Tools and connections](/docs/desktop/user-guide/tools-and-connections) |

The default agent "Ryu" manages its own Pi binary, so it is self-sufficient. Every other built-in
agent (Claude Code, Codex, Gemini CLI, bare Pi, OpenClaw) is opt-in via the agents catalog.
Onboarding detects which agent CLIs you already have on PATH and offers to add the matching agent
through `GET /api/agents/catalog` and `POST /api/agents/catalog/{install,uninstall}`. The picker
at `GET /api/agents` only lists the installed set.

## Modalities and standards

All four modalities are first-class and swappable: chat, image generation, voice TTS, and STT.
Image generation and TTS playback backends exist but are not yet surfaced in the desktop chat UI,
where only STT voice input is wired today.

Ryu treats the Agent **Skills** standard, **MCP**, and **ACP** as first-class. A fresh install can
already read installed skills and call MCP tools through the unified registry.

<Callout type="info">
  The default agent fails to start until the managed Pi engine is installed, which onboarding
  warms but does not always install end to end on every machine. If "Ryu" reports that `pi` was
  not found, install it from the agents catalog or pick another installed agent.
</Callout>
